Como testar rapidamente
1. Backend:
   Copy-Item backend\.env.example backend\.env
   # editar backend\.env com suas chaves locais
   cd backend; npm install; npm run dev
   Testar endpoints: POST /redacoes (imagem/base64), GET /redacoes/:id/analise-enem

2. Frontend:
   cd frontend; npm install; npm start
   No Dashboard: upload -> Enviar -> abrir Análise

Notas:
- O template de .env está em backend/.env.example. Não comite .env com chaves.
- Se ocorrerem problemas com o DB local (backend/prisma/dev.db), favor renomear/backup antes do merge.
